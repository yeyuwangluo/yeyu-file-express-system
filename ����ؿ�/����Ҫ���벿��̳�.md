# 叶宇文件快递部署要求与部署教程

本文适用于源码包形式部署叶宇文件快递系统。项目基于 Laravel 9.52，前端静态资源已预编译，部署时通常不需要 Node.js。

## 1. 源码包说明

源码包应包含：

- `app/`、`config/`、`database/`、`routes/`、`resources/` 等 Laravel 源码
- `public/` 静态资源和入口文件
- `composer.json`、`composer.lock`
- `.env.example`
- `docs/` 文档

源码包不应包含：

- `.env`
- `vendor/`
- `node_modules/`
- 上传文件、运行日志、缓存文件
- 服务器密码、数据库密码、后台账号密码等敏感信息

## 2. 服务器要求

推荐环境：

- Linux 服务器，推荐使用宝塔面板、1Panel、LNMP 或手动 Nginx/PHP-FPM 环境
- PHP 8.0.2 或以上，推荐 PHP 8.1 / 8.2
- Composer 2
- MySQL 5.7+ / MySQL 8.0+，也可使用 SQLite
- Nginx 或 Apache
- 网站运行目录必须指向项目的 `public` 目录

必需 PHP 扩展：

```text
fileinfo
openssl
pdo_mysql
mbstring
tokenizer
xml
ctype
json
curl
zip
```

建议 PHP 配置：

```ini
upload_max_filesize = 50M
post_max_size = 50M
max_execution_time = 120
memory_limit = 256M
```

如果要支持更大的上传文件，需要同时调整：

- `.env` 中的 `UPLOAD_MAX_FILE_SIZE`
- PHP 的 `upload_max_filesize`、`post_max_size`
- Nginx 的 `client_max_body_size`
- CDN 的上传大小限制

## 3. 宝塔面板可视化部署教程

### 3.1 创建站点

1. 登录宝塔面板。
2. 打开“网站”。
3. 点击“添加站点”。
4. 填写域名，例如 `file.example.com`。
5. PHP 版本选择 PHP 8.0、8.1、8.2 或更高。
6. 数据库选择 MySQL，并创建数据库。
7. 提交创建。

### 3.2 上传源码

1. 打开宝塔“文件”。
2. 进入站点目录，例如：

```text
/www/wwwroot/file.example.com
```

3. 上传源码压缩包。
4. 解压源码包。
5. 确认目录中能看到 `artisan`、`composer.json`、`public`、`app` 等文件或目录。

如果解压后多了一层目录，例如：

```text
/www/wwwroot/file.example.com/xiaoxin-file-express-system
```

后续命令中的项目目录就使用这个路径。

### 3.3 设置运行目录

在宝塔站点设置中打开“网站目录”，把运行目录设置为项目的 `public`：

```text
/public
```

如果源码保留在子目录中，则填写：

```text
/xiaoxin-file-express-system/public
```

注意：网站根目录不能直接指向项目根目录，否则 `.env`、源码和依赖文件存在暴露风险。

### 3.4 配置伪静态

打开站点设置里的“伪静态”，选择 Laravel 模板。若没有模板，粘贴：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

保存后重载 Nginx。

### 3.5 安装 PHP 扩展

打开“软件商店”，找到当前使用的 PHP 版本，进入“安装扩展”，确认安装：

```text
fileinfo
openssl
pdo_mysql
mbstring
curl
zip
```

`json`、`ctype`、`tokenizer`、`xml` 通常已内置。如果安装依赖或上传文件时报错，优先检查 `fileinfo` 和 `pdo_mysql`。

### 3.6 安装 Composer 依赖

打开宝塔“终端”，进入项目目录：

```bash
cd /www/wwwroot/file.example.com
```

如果源码在子目录：

```bash
cd /www/wwwroot/file.example.com/xiaoxin-file-express-system
```

执行：

```bash
composer install --no-dev --prefer-dist --optimize-autoloader
```

不要在生产服务器执行 `composer update`，应使用源码包自带的 `composer.lock`。

如果 Composer 报错找不到开发包，可以先清理 Laravel 缓存：

```bash
rm -f bootstrap/cache/*.php
composer dump-autoload --optimize --no-dev
```

### 3.7 配置目录权限

在终端执行：

```bash
chmod -R 775 storage bootstrap/cache
chown -R www:www storage bootstrap/cache
```

如果服务器 Web 用户不是 `www`，请按实际用户调整。

### 3.8 初始化系统

方式一：使用安装页面。

1. 访问你的域名。
2. 未安装时系统会自动跳转到 `/install`。
3. 按页面填写数据库、站点地址、管理员账号和密码。
4. 提交后系统会自动写入 `.env`、生成密钥、迁移数据库并创建管理员。

方式二：手动命令初始化。

```bash
cp .env.example .env
php artisan key:generate --force
php artisan migrate --force
php artisan db:seed --force
php artisan storage:link
```

手动方式需要先编辑 `.env`，至少配置：

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://file.example.com
XIAOXIN_FILE_EXPRESS_INSTALLED=true

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=你的数据库名
DB_USERNAME=你的数据库用户名
DB_PASSWORD=你的数据库密码

ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=请改成强密码
```

### 3.9 生产缓存优化

安装成功后执行：

```bash
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

后续修改 `.env` 后，也需要重新执行：

```bash
php artisan optimize:clear
php artisan config:cache
```

### 3.10 配置计划任务

宝塔“计划任务”添加 Shell 脚本：

- 周期：每分钟
- 脚本内容：

```bash
cd /www/wwwroot/file.example.com && php artisan schedule:run >> /dev/null 2>&1
```

如果项目在子目录，请把路径改成真实项目目录。

### 3.11 配置队列任务

正式环境建议使用 Supervisor。启动命令：

```bash
php artisan queue:work database --sleep=3 --tries=3 --timeout=120
```

如果没有 Supervisor，可在宝塔计划任务中每分钟执行一次：

```bash
cd /www/wwwroot/file.example.com && php artisan queue:work database --once --tries=3 --timeout=120 >> /dev/null 2>&1
```

## 4. 命令行部署教程

```bash
cd /var/www
unzip xiaoxin-file-express-system.zip
cd xiaoxin-file-express-system

composer install --no-dev --prefer-dist --optimize-autoloader
cp .env.example .env
php artisan key:generate --force
php artisan migrate --force
php artisan db:seed --force
php artisan storage:link
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Nginx 示例：

```nginx
server {
    listen 80;
    server_name file.example.com;
    root /var/www/xiaoxin-file-express-system/public;
    index index.php index.html;

    client_max_body_size 50m;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

不同系统的 PHP-FPM socket 可能不同，例如：

```text
/run/php/php8.0-fpm.sock
/run/php/php8.1-fpm.sock
/run/php/php8.2-fpm.sock
127.0.0.1:9000
```

请按实际环境修改。

## 5. CDN 与 HTTPS 注意事项

如果域名前面套 CDN：

- CDN 回源地址填写源站 IP 或源站域名
- 回源 Host 填写真实站点域名，例如 `file.example.com`
- 如果源站只配置了 HTTP，CDN 回源协议请选择 HTTP
- 如果源站配置了 HTTPS，CDN 可以选择 HTTPS 或协议跟随
- CDN 不建议缓存 `/api/*`、`/admin*`、`/files/*`
- 静态资源可以缓存，例如 `/_next/static/*`、`/css/*`、`/fonts/*`
- 修改部署后要刷新 CDN 缓存
- 上传失败时检查 CDN 的请求体大小限制

推荐 CDN 缓存规则：

```text
不缓存：/api/*
不缓存：/admin*
不缓存：/files/*
缓存：/_next/static/*
缓存：/css/*
缓存：/fonts/*
```

## 6. 后台入口

后台入口：

```text
/admin
```

系统会跳转到：

```text
/admin-lite
```

首次登录后请立即修改管理员邮箱和密码。

## 7. 常见问题

### 首页 403 或目录列表

网站运行目录没有指向 `public`。请在 Web 服务器或宝塔中把运行目录改为项目的 `public`。

### 页面 404

通常是伪静态没有配置 Laravel 规则：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

### 安装页提示目录不可写

执行：

```bash
chmod -R 775 storage bootstrap/cache
chown -R www:www storage bootstrap/cache
```

### 上传失败

依次检查：

- PHP `upload_max_filesize`
- PHP `post_max_size`
- Nginx `client_max_body_size`
- CDN 上传大小限制
- 后台上传大小配置
- `storage` 目录权限

### 修改 `.env` 后不生效

执行：

```bash
php artisan optimize:clear
php artisan config:cache
```

### Composer 安装失败

检查 PHP 版本和扩展：

```bash
php -v
php -m
```

确认启用 `fileinfo`、`pdo_mysql`、`openssl`、`mbstring`、`curl`、`zip`。

### CDN 访问还是旧页面

检查：

- CDN 回源 IP 是否正确
- 回源 Host 是否正确
- 回源协议是否与源站一致
- 是否已经刷新 CDN 缓存

## 8. 上线验收清单

- 首页可以打开，样式正常
- `/terms` 协议页可以打开并能点击同意
- `/admin` 可以进入后台
- 能上传文件
- 分享页能打开
- 提取码下载正常
- 分片上传正常
- 局域网互传页面可访问
- 后台能修改设置、管理员账号、公告、协议内容
- 计划任务和队列任务正常运行
- 生产环境 `APP_DEBUG=false`
