# 叶宇文件快递部署教程

本文档适用于源码包 `xiaoxin-file-express-system.zip`。系统是 Laravel 9.52 项目，兼容 PHP 8.0.2+，数据库兼容 MySQL 5.7+。

> 前端 CSS/JS 已预编译到 `public/_next/static/` 目录，部署时不需要 Node.js。

## 一、源码包内容

源码目录名为 `xiaoxin-file-express-system`。

源码包已包含：

- Laravel 后端源码
- 预编译的前端 CSS/JS（`public/_next/static/`）
- 预编译的静态页面（首页、互传页、状态页、协议页、App 页）
- `composer.lock`
- `.env.example`
- 数据库迁移和安装页面

源码包不包含：

- `.env`
- `vendor`
- 本地上传文件、日志、缓存

## 二、服务器要求

- PHP 8.0.2 或以上（推荐 8.1 或 8.2）
- MySQL 5.7+ 或 SQLite
- Nginx 或 Apache
- PHP 扩展：`fileinfo`、`openssl`、`pdo_mysql`、`mbstring`、`tokenizer`、`xml`、`ctype`、`json`、`curl`

**不需要：** Node.js、npm、Redis（可选）

普通虚拟主机能否部署，取决于是否支持：

- 网站运行目录指向 `public`
- Composer 安装依赖（或本地打包好 `vendor/` 后上传）
- 执行 `php artisan migrate`
- 设置目录写入权限

如果虚拟主机不支持这些能力，建议使用宝塔面板服务器或 VPS。

## 三、宝塔面板可视化部署

### 1. 创建站点

1. 登录宝塔面板。
2. 打开"网站"。
3. 点击"添加站点"。
4. 域名填写你的域名，例如 `example.com`。
5. PHP 版本选择 `8.0`、`8.1`、`8.2` 或更高。
6. 数据库选择 MySQL，数据库名建议 `xiaoxin_file_express`。
7. 提交创建。

### 2. 上传源码

1. 打开宝塔"文件"。
2. 进入站点目录，例如 `/www/wwwroot/example.com`。
3. 上传 `xiaoxin-file-express-system.zip`。
4. 解压。
5. 解压后应看到目录 `/www/wwwroot/example.com/xiaoxin-file-express-system`。

也可以把 `xiaoxin-file-express-system` 目录内的文件移动到站点根目录，但后续教程默认保留这个目录。

### 3. 设置网站运行目录

1. 打开宝塔"网站"。
2. 找到站点，点击"设置"。
3. 打开"网站目录"。
4. 运行目录选择或填写：

```text
/xiaoxin-file-express-system/public
```

5. 保存。

最终 Web 根目录必须指向 Laravel 的 `public` 目录，不能指向项目根目录。

### 4. 配置伪静态

在宝塔站点设置里打开"伪静态"，选择 Laravel。如果没有 Laravel 模板，粘贴：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

保存后重载 Nginx。

### 5. 安装 PHP 扩展

1. 打开宝塔"软件商店"。
2. 找到当前站点使用的 PHP 版本。
3. 点击"设置"。
4. 打开"安装扩展"。
5. 确认安装或启用：

```text
fileinfo
openssl
pdo_mysql
mbstring
tokenizer
xml
curl
```

`json` 和 `ctype` 通常已内置。

### 6. 配置 PHP 上传限制

在 PHP 设置里调整：

```ini
upload_max_filesize = 50M
post_max_size = 50M
max_execution_time = 120
memory_limit = 256M
```

如果要上传更大的文件，同时还要调整 Nginx 的 `client_max_body_size`。

### 7. 安装 Composer 依赖

打开宝塔"终端"，进入项目目录：

```bash
cd /www/wwwroot/example.com/xiaoxin-file-express-system
```

执行：

```bash
composer install --no-dev --optimize-autoloader
```

不要执行 `composer update`，生产环境应使用源码包里的 `composer.lock`。

如果宝塔终端提示没有 Composer，可以在宝塔"软件商店"安装 Composer，或用：

```bash
php /www/server/php/你的PHP版本/bin/composer install --no-dev --optimize-autoloader
```

具体路径以宝塔实际 PHP 版本为准。

**虚拟主机用户：** 如果虚拟主机没有 Composer，可以在自己电脑上先执行 `composer install --no-dev --optimize-autoloader`，生成 `vendor/` 目录后，把整个项目（含 `vendor/`）一起上传到虚拟主机。

### 8. 配置目录权限

在宝塔"文件"中确认以下目录可写：

```text
storage
bootstrap/cache
database
```

Linux 服务器可在终端执行：

```bash
chmod -R 775 storage bootstrap/cache database
chown -R www:www storage bootstrap/cache database
```

如果服务器 Web 用户不是 `www`，按实际用户调整。

### 9. 通过安装页面完成初始化

1. 浏览器打开你的域名。
2. 如果系统未安装，会自动跳转到：

```text
/install
```

3. 按页面提示填写：

- 站点名称：叶宇文件快递
- 站点地址：你的域名
- 数据库类型：MySQL 或 SQLite
- 数据库地址：通常是 `127.0.0.1`
- 数据库名、用户名、密码：宝塔创建站点时生成的数据库信息
- 管理员账号和密码

4. 提交安装。

安装程序会自动：

- 写入 `.env`
- 生成 `APP_KEY`
- 执行数据库迁移
- 写入默认配置
- 创建管理员账号
- 写入安装标记

### 10. 创建存储链接

安装完成后，在宝塔终端执行：

```bash
cd /www/wwwroot/example.com/xiaoxin-file-express-system
php artisan storage:link
```

如果提示链接已存在，可以忽略。

### 11. 生产缓存优化

确认安装成功后执行：

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

如果后续修改 `.env`，需要执行：

```bash
php artisan optimize:clear
php artisan config:cache
```

### 12. 配置计划任务

打开宝塔"计划任务"，添加 Shell 脚本任务：

- 任务名称：叶宇文件快递定时任务
- 执行周期：每分钟
- 脚本内容：

```bash
cd /www/wwwroot/example.com/xiaoxin-file-express-system && php artisan schedule:run >> /dev/null 2>&1
```

这个任务负责过期文件清理、统计生成、日志清理、备份等。

### 13. 配置队列任务

如果宝塔已安装 Supervisor 管理器，新增一个进程：

- 名称：`xiaoxin-file-express-queue`
- 运行目录：`/www/wwwroot/example.com/xiaoxin-file-express-system`
- 启动命令：

```bash
php artisan queue:work database --sleep=3 --tries=3 --timeout=120
```

如果没有 Supervisor，可以用宝塔计划任务每分钟执行一次作为替代：

```bash
cd /www/wwwroot/example.com/xiaoxin-file-express-system && php artisan queue:work database --once --tries=3 --timeout=120 >> /dev/null 2>&1
```

Supervisor 更适合正式生产环境。

## 四、虚拟主机部署

虚拟主机通常没有 SSH 和 Composer，需要在本地打包好再上传。

### 本地打包步骤

```bash
cd xiaoxin-file-express-system
composer install --no-dev --optimize-autoloader
```

然后把整个项目目录（含 `vendor/`）压缩成 zip，上传到虚拟主机解压。

### 虚拟主机上完成部署

1. 确认网站运行目录指向 `public`
2. 配置伪静态（Laravel 规则）
3. 设置 `storage`、`bootstrap/cache`、`database` 目录可写
4. 访问 `/install` 完成安装

### 虚拟主机的限制

虚拟主机无法运行常驻进程，以下功能受限：

- 队列任务（SHA-256 计算、病毒扫描、文件删除）不自动执行
- 定时任务（过期清理、统计、备份）可能不自动执行

如果虚拟主机支持 cron，添加以下计划任务可恢复大部分自动化：

```cron
* * * * * cd /你的网站目录 && php artisan schedule:run >> /dev/null 2>&1
* * * * * cd /你的网站目录 && php artisan queue:work --stop-when-empty --max-time=60 >> /dev/null 2>&1
```

核心文件分享功能（上传、下载、提取码、分片上传、局域网互传、后台管理）不受影响。

## 五、命令行部署步骤

如果不用宝塔，可以按下面流程部署：

```bash
cd /var/www
unzip xiaoxin-file-express-system.zip
cd xiaoxin-file-express-system
composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate --force
php artisan migrate --force
php artisan storage:link
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Nginx 站点根目录指向：

```text
/var/www/xiaoxin-file-express-system/public
```

Nginx 规则：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

## 六、后台入口

后台地址：

```text
/admin
```

系统会跳转到：

```text
/admin-lite
```

管理员账号使用安装页面创建的账号。首次登录后建议立即修改密码。

## 七、常见问题

### 1. 打开首页显示 403 或目录列表

网站运行目录没有指向 `public`。请在宝塔"网站目录"中设置运行目录为：

```text
/xiaoxin-file-express-system/public
```

### 2. 打开页面 404

伪静态没有配置 Laravel 规则。设置：

```nginx
location / {
    try_files $uri $uri/ /index.php?$query_string;
}
```

### 3. 安装页提示目录不可写

给目录写入权限：

```bash
chmod -R 775 storage bootstrap/cache database
chown -R www:www storage bootstrap/cache database
```

### 4. 上传文件失败

检查：

- PHP `upload_max_filesize`
- PHP `post_max_size`
- Nginx `client_max_body_size`
- 后台上传大小配置
- `storage` 目录权限

### 5. Composer 安装失败

确认 PHP 版本和扩展：

```bash
php -v
php -m
```

必须启用 `fileinfo`、`pdo_mysql`、`openssl`、`mbstring`、`tokenizer`、`xml`、`curl`。

### 6. 修改 `.env` 后不生效

执行：

```bash
php artisan optimize:clear
php artisan config:cache
```

### 7. 页面样式异常

确认 `public/_next/static/css/` 目录存在且文件完整。如果文件缺失，重新上传源码包。

## 八、上线验收清单

- 首页可打开，样式正常
- 未安装时能自动进入 `/install`
- 安装流程能完成
- `/admin` 后台能登录
- 能上传文件
- 分享链接能打开
- 提取码下载正常
- 局域网互传页面可访问
- 服务状态页面可访问
- 过期文件能被计划任务清理
- 宝塔计划任务每分钟正常执行
- 队列任务正常运行
